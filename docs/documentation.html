BookWise Documentation

BookWise is a modern personal library management app that lets users organize, review, and explore their book collection with ease. It features category management, in-app reading for digital books (EPUB, PDF, and more), book cover uploads, and secure user authentication. Built with Node.js, Express, Handlebars, Sequelize, SQLite, and Tailwind CSS v3.

Getting Started

  1. Clone the repository: 
    git clone https://github.com/your-username/bookwise.git
cd bookwise
  2. Install dependencies: 
    npm install
  3. Set up your database:
    • Configure your database in config/config.json (default is SQLite).
    • Run migrations: 
      npx sequelize-cli db:migrate
  4. (Optional) Seed the database with demo data: 
    npx sequelize-cli db:seed:all
  5. Build Tailwind CSS v3: 
    npx tailwindcss -i ./public/styles/tailwind.css -o ./public/styles/output.css --watch
  6. Run the app: 
    npm start
    Visit http://localhost:3000 in your browser.

Project Structure Update: Layouts and Modals

Summary: The project structure was updated for clarity and maintainability. The views/layouts/ folder now contains the main layout file (layout.hbs) and layout partials (header.hbs, footer.hbs). A modals/ folder was also created inside views/partials/ to organize all modal partials. 


bookwise/
|-- views/
|   |-- layouts/           # Layouts and layout partials
|   |   |-- layout.hbs     # Main layout file (with {{{body}}})
|   |   |-- header.hbs     # Header partial
|   |   |-- footer.hbs     # Footer partial
|   |-- partials/
|   |   |-- modals/        # Modal partials (authModal.hbs, aboutModal.hbs, etc.)
|   |   |-- navbar.hbs     # Other partials
|   |-- dashboard.hbs      # Dashboard main view
|   |-- landing.hbs        # Landing page main view
|   |-- ...                # Other main views

How to register and use layouts and partials in app.js: 


const hbs = require('hbs');
const path = require('path');

// Register partials (for header, footer, modals, etc.)
hbs.registerPartials(path.join(__dirname, 'views', 'partials'));
hbs.registerPartials(path.join(__dirname, 'views', 'layouts'));

app.set('views', path.join(__dirname, 'views'));
app.set('view engine', 'hbs');

// Set the default layout to use views/layouts/layout.hbs
app.set('view options', { layout: 'layouts/layout' });

Usage: Your main layout file should be at views/layouts/layout.hbs and contain {{{body}}} where page content will be injected. In your views, you can include layout partials with {{> header}} and {{> footer}}, and modal partials with {{> modals/aboutModal}}.

Usage Guide

  • User Authentication: Sign up and log in to access your personal library. Passwords are securely hashed.
  • Managing Books: Add, edit, or delete books. Assign books to categories and upload cover images or provide image URLs.
  • Categories: Create, edit, and delete categories. Filter your library by category.
  • In-App Reading: Open EPUB, PDF, or plain text books directly in the app. (Uses epub.js and PDF.js.)
  • Sticky Notes: (Planned) Add sticky notes or highlights to specific pages while reading.
  • Search & Filter: Use the search bar to find books by title, author, or category.
  • Profile & Dashboard: View and edit your profile, see library stats, and manage your collection from a responsive dashboard.

Troubleshooting

  • Tailwind CSS not building? Ensure you’re using Node.js v16.13.0 or higher and have installed all dependencies. See Tailwind CSS docs.
  • Database errors? Check your config/config.json and ensure migrations have been run.
  • Authentication issues? Ensure session or JWT configuration is correct and passwords are hashed.
  • Need more help? Open an issue on the Issue Tracker.
